## Fixes aplicados nesta sessão

### inventory-manager.tsx — PriceModal

- `useDialog(true, onClose)` hook aplicado: Escape fecha, foco inicial no input, trap entre focáveis, foco anterior restaurado
- `role="dialog"` + `aria-modal="true"` + `aria-label={Preço customizado de ${nome}}`
- Overlay `<div onClick>` marcado `aria-hidden="true"`
- Close button ganha `type="button"` + `aria-label="Fechar"` + `title="Fechar (Esc)"`
- SVG do X marcado `aria-hidden="true"`
- `@keyframes scaleIn` inline via `<style jsx>` **removido** → classe `fade-in` do `globals.css` (motion namespace uno)
- Surfaces alinhadas com DS: `bg-zinc-950` → `bg-[#1a1f2e]`, `border-zinc-800` → `border-white/[0.14]`
- Botão "Limpar" migrou `red-*` → `rose-*` (token DS §2.3)

### inventory-manager.tsx — StickerCard + grid

- **11 `<button>`s ganharam `type="button"`** (lines ~222, 264, 309, 316, 701, 729, 780, 842, 850, 884, 900)
- Botão de preço customizado: `opacity-0 group-hover:opacity-100` → `opacity-70 group-hover:opacity-100` — touch devices agora veem o botão com 70% opacidade por default, hover desktop leva a 100%
- Input de busca: `aria-label="Buscar figurinha"` (WCAG 1.3.1 + 3.3.2)

## Findings não críticos deixados como-estão

- **Radius `rounded-2xl`** no PriceModal — declarado como pattern no núcleo (ConfirmDialog usa o mesmo). Não é slop.
- **`bg-[#1a1f2e]`** como surface de modal — segue o DS §2.1 surface "elevated". OK.
- **Arbitrary font-sizes `text-[7px]/[10px]/[11px]`** em badges e metadata — escala micro não tokenizada no DS §3. Já presente em dezenas de componentes. Ver gap conhecido.
- **Breakpoints e surface patterns** do dashboard seguem o core — sem divergência material.
