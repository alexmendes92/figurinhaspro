## Findings críticos (8)

### Axis 1 — System conformance (4)

**`surfaces-off-token` · inventory-manager.tsx:46**
Modal usa `bg-zinc-950` (#09090b) como surface. `DESIGN_SYSTEM.md §2.1` declara `#0d1017 / #0f1219 / #111318 / #1a1f2e` como surfaces canônicas — `zinc-950` não está no ladder.
→ Trocar por surface declarada (`#1a1f2e` para modal elevado sobre canvas escuro).

**`surfaces-off-token` · inventory-manager.tsx:197**
StickerCard usa `bg-zinc-900` (#18181b) com variações de opacidade. DS declara card/panel surface = `#0f1219` — off-palette de ~5%.
→ Substituir por `bg-[#0f1219]` (card) ou `bg-[#1a1f2e]` (elevated).

**`feedback-color-off-token` · inventory-manager.tsx:232**
Success color usa `green-500` (#22c55e) ao invés do token emerald declarado em DS §2.3 (success = `#34d399`). Repetido em linhas 247, 841, 874.
→ Padronizar com `bg-emerald-400`.

**`feedback-color-off-token` · inventory-manager.tsx:849**
Error/destructive usa `red-500/red-400`. DS §2.3 declara error = `#f87171` (rose). `red-500` (#ef4444) é divergente. Repetido em linhas 866, 874, 875, 899.
→ Padronizar com `rose-400` ou arbitrary `#f87171`.

### Axis 2 — Accessibility (4)

**`modal-no-escape-close` · inventory-manager.tsx:42**
PriceModal não fecha com tecla Escape. Viola WCAG 2.1.2 (No Keyboard Trap) — usuário de teclado pode ficar preso.
→ `useEffect` com `window.addEventListener("keydown", ...)` escutando Escape.

**`div-with-click-handler` · inventory-manager.tsx:44**
Overlay do modal é `<div>` com `onClick` mas sem `role`, `tabIndex` nem keyboard handler. Acoplado ao Escape ausente, modal fica inoperável por teclado exceto pelo botão X.
→ Escape resolve — overlay pode ficar só decorativo.

**`touch-unreachable-affordance` · inventory-manager.tsx:265**
Botão `

 (preço custom) fica `opacity-0` até hover quando não há preço. Touch devices não têm hover estável — affordance inacessível em mobile. `/painel` é mobile-first declarado em CLAUDE.md. **Bloqueante.**
→ Tornar sempre visível (opacity sutil) OU usar `matchMedia("(hover: hover)")` para hide-on-hover só em dispositivos com hover.

**`missing-accessible-name` · inventory-manager.tsx:73**
Botão de fechar modal (ícone X) sem `aria-label`, `title` ou texto. Screen readers anunciam apenas "botão". Viola WCAG 4.1.2.
→ `aria-label="Fechar"` + `title="Fechar (Esc)"` após implementar Escape.
